## lang атрибут

`lang` — один из самых важных атрибутов с точки зрения доступности и i18n.

```html
<!-- На элементе <html>: основной язык документа -->
<html lang="ru">

<!-- Вариант с регионом: язык + регион -->
<html lang="ru-RU">
<html lang="en-US">
<html lang="zh-CN"> <!-- Упрощённый китайский -->
<html lang="zh-TW"> <!-- Традиционный китайский -->
<html lang="pt-BR"> <!-- Португальский, Бразилия -->

<!-- Inline: изменение языка для части текста -->
<p>
  Английское слово
  <span lang="en">frontend</span>
  широко используется в индустрии.
</p>

<!-- Для элементов с иностранным текстом -->
<blockquote lang="en">
  <p>Any fool can write code that a computer can understand.
  Good programmers write code that humans can understand.</p>
  <footer>Martin Fowler</footer>
</blockquote>
```

### Зачем `lang` на senior-уровне

- **Screen readers**: используют `lang` для выбора голоса и произношения. Без `lang` screen reader будет произносить русский текст с акцентом языка системы.
- **CSS `hyphens`**: автоматические переносы зависят от `lang`.
- **Spell check**: браузерная проверка орфографии.
- **Search engines**: индексирование с учётом языка.
- **Quote marks**: `<q>` использует `lang` для выбора кавычек.
- **WCAG 3.1.1** (A): язык страницы обязателен.
- **WCAG 3.1.2** (AA): части страницы на другом языке должны быть помечены.

## dir — направление текста

```html
<!-- На <html>: базовое направление всего документа -->
<html lang="ar" dir="rtl">  <!-- Арабский: справа налево -->
<html lang="he" dir="rtl">  <!-- Иврит: справа налево -->
<html lang="ru" dir="ltr">  <!-- Русский: слева направо (default) -->

<!-- Inline изменение направления -->
<p dir="rtl">هذا نص عربي</p>

<!-- auto: браузер определяет по первому символу с явным направлением -->
<p dir="auto">{{ user_content }}</p>
```

### RTL в CSS

```css
/* Логические свойства — работают с любым dir -->
.element {
  margin-inline-start: 1rem;  /* Вместо margin-left/right в зависимости от dir */
  padding-inline-end: 1rem;
  border-inline-start: 2px solid;
  inset-inline-start: 0;
}

/* Флип для RTL */
[dir="rtl"] .icon {
  transform: scaleX(-1); /* Стрелки, chevron'ы — нужно зеркалить */
}
```

## Unicode Bidirectional Algorithm

Алгоритм UAX #9, встроенный в браузеры, определяет как отображать смешанный текст.

### Проблема: BiDi спуфинг

```html
<!--
  Пример уязвимости: имя файла выглядит как "image.png"
  но реально называется "gnp.egami" из-за RTL символа
  
  U+202E (RIGHT-TO-LEFT OVERRIDE) переключает направление.
  Файл: malicious‮exe.nip  → отображается как: pin.exe
  
  Эта уязвимость использовалась для скрытия вредоносных расширений файлов.
  GitHub и другие платформы экранируют эти символы.
-->
```

### Элементы для управления BiDi

```html
<!--
  <bdi> (Bidirectional Isolation):
  Изолирует направление текста от окружения.
  Критично для пользовательского контента с неизвестным направлением.
-->
<p>
  Пользователь
  <!-- Без <bdi>: арабское имя может сломать пунктуацию вокруг -->
  <bdi>محمد علي</bdi>
  написал сообщение.
</p>

<!-- Список пользователей с разными именами -->
<ul>
  <li>Место 1: <bdi>Иван</bdi></li>
  <li>Место 2: <bdi>محمد</bdi></li>
  <li>Место 3: <bdi>王芳</bdi></li>
</ul>

<!--
  <bdo> (Bidirectional Override):
  Явно переопределяет направление. Редко используется.
-->
<p><bdo dir="rtl">Этот текст будет отображён справа налево</bdo></p>
```

> ⚠️ **Ловушка:** Всегда оборачивай пользовательские данные (имена, адреса) в `<bdi>` в многоязычных приложениях. Арабское или ивритское имя без `<bdi>` может сломать окружающий текст: «Место 1: محمد: баллов» отобразится с неправильным порядком двоеточий.

## hreflang

Подробно в [05-seo-meta](../05-seo-meta/README.md#hreflang--мультиязычные-сайты).

```html
<!-- Коды: RFC 5646 -->
<!-- Только язык: -->  <link rel="alternate" hreflang="ru">
<!-- Язык + регион: --> <link rel="alternate" hreflang="en-US">
<!-- Default: -->       <link rel="alternate" hreflang="x-default">
```

## translate атрибут

Подсказка для инструментов автоперевода (Google Translate и т.д.):

```html
<!-- translate="no": не переводить -->
<span translate="no">Kubernetes</span>
<code translate="no">npm install</code>
<span translate="no" lang="en">frontend</span>

<!-- Имена брендов, технические термины -->
<p>
  Мы используем <span translate="no">React</span> для разработки.
</p>

<!-- translate="yes": переводить (по умолчанию) -->
<p translate="yes">Этот текст следует перевести.</p>

<!-- На родительском элементе: наследуется -->
<article translate="no">
  <!-- Весь код в статье не будет переводиться -->
  <pre><code>const x = 1;</code></pre>
</article>
```

## Специфичные элементы i18n

### `<ruby>` — фонетические аннотации

Используется для иероглифических языков (японский, китайский):

```html
<!-- ruby: фонетика над иероглифами -->
<ruby>
  漢 <rt>かん</rt>
  字 <rt>じ</rt>
</ruby>

<!-- с fallback для браузеров без поддержки ruby -->
<ruby>
  漢字
  <rp>(</rp>
  <rt>かんじ</rt>
  <rp>)</rp>
</ruby>
```

### Числа, даты, валюта — через JavaScript

HTML не имеет встроенных элементов для локализации чисел, дат и валют. Используй `Intl` API:

```javascript
// Форматирование с учётом локали
const price = new Intl.NumberFormat('ru-RU', {
  style: 'currency',
  currency: 'RUB',
}).format(1234567.89);
// → "1 234 567,89 ₽"

const date = new Intl.DateTimeFormat('ru-RU', {
  year: 'numeric',
  month: 'long',
  day: 'numeric',
}).format(new Date('2025-01-15'));
// → "15 января 2025 г."
```

```html
<!-- <time> для машиночитаемых дат (локализованный текст внутри) -->
<time datetime="2025-01-15">
  15 января 2025 г.
</time>
```

## Вопросы на интервью

1. **Почему `lang` важен для screen reader?**
   > Screen reader использует `lang` для выбора движка TTS (Text-to-Speech). Без `lang` русский текст будет читаться с акцентом языка по умолчанию. Пометка `lang="en"` на английских словах внутри русского текста — WCAG 3.1.2 (AA).

2. **В чём разница между `<bdi>` и `<bdo>`?**
   > `<bdi>` изолирует контент — направление определяется автоматически по содержимому, не влияет на окружающий текст. `<bdo>` принудительно переопределяет направление. `<bdi>` для пользовательского контента, `<bdo>` — редко.

3. **Что такое BiDi спуфинг и как с ним бороться?**
   > Unicode содержит символы переключения направления (U+202E и другие). Злоумышленник может назвать файл так, что его расширение будет выглядеть иначе. Защита: экранировать или удалять BiDi символы при обработке пользовательского ввода.

4. **Когда `lang="en"` не достаточно и нужно `lang="en-US"`?**
   > Когда важен регион: орфография (colour vs color), форматирование дат (MM/DD/YYYY vs DD/MM/YYYY), специфичное произношение. Для `hreflang` — чтобы Google показывал правильную версию пользователям из конкретной страны.
