Demo 3: z-index ловушка — Stacking Context

Синий (z-index:9999) перекрывается зелёным (z-index:1)? Да! Потому что синий внутри SC с z-index:1, а зелёный — в SC с z-index:2.

SC z-index:1
z-index: 9999 (внутри SC z-index:1)
SC z-index:2 — перекрывает весь первый SC
z-index: 1 (внутри SC z-index:2)

Вывод: z-index соревнуется только внутри одного Stacking Context. SC z-index:2 целиком выше SC z-index:1 вне зависимости от дочерних z-index значений.

Demo 4: isolation: isolate — изолируй компонент

isolation: isolate — z-index внутри не конкурирует с внешними элементами
z-index: 999 — только внутри своего SC

Используй isolation: isolate на модальных окнах, дропдаунах, тостах — чтобы внутренние z-index не конфликтовали с глобальными.

Demo 5: IFC баг — gap под inline изображением

❌ inline img: gap снизу (красный фон видно)

✅ display: block на img: нет gap

Inline элемент выравнивается по baseline строки. Браузер резервирует место для descenders (g, y, p) ниже baseline. Это gap — не padding, не border, не margin. Решение: display: block, vertical-align: bottom, или line-height: 0 на родителе.
