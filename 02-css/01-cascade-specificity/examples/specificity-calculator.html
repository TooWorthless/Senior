Demo 1: Базовое вычисление

Селектор A (ID) B (Class/Attr/Pseudo-class) C (Element/Pseudo-el) Итог

Demo 2: :is() «заражает» специфичностью наибольшего аргумента

СелекторСпецифичностьПримечание
:is(#id, .class) p (1,0,1) :is() берёт (1,0,0) из #id + p = (0,0,1)
.class p (0,1,1) Ожидаемо

⚠️ :is(#id, .class) p имеет специфичность (1,0,1) даже для элементов без #id родителя.

Demo 3: :where() — нулевая специфичность (легко переопределить)

СелекторСпецифичность
:where(#demo3 .btn) (0,0,0) — :where обнуляет
.btn-override (0,1,0) — победитель

Применение: библиотеки и reset-стили через :where() — пользователь легко переопределит с минимальной специфичностью.

Demo 5: :not() берёт специфичность своего аргумента

СелекторСпецифичностьРасчёт
p:not(#special) (1,0,1) p=(0,0,1) + :not(#special)→(1,0,0) = (1,0,1)
p.regular (0,1,1) p=(0,0,1) + .regular=(0,1,0)

Оранжевый: p:not(#special) победил, несмотря на то что этот <p> не является #special

Это #special — тоже оранжевый, т.к. p:not(#special) не совпадает, применяется p.regular
